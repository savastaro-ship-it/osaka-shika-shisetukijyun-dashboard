# 大阪府 歯科 施設基準 届出状況ダッシュボード

近畿厚生局が公開している「施設基準の届出受理状況（歯科）」をもとに、
大阪府内の歯科医療機関の **届出状況と届出率** を表示する静的ダッシュボードです。
GitHub Actions が毎日1回データ更新をチェックし、新しい月のデータが出たら自動で反映します。

## 構成

```
.
├── index.html                 # ダッシュボード本体（軽量・自己完結）
├── data/
│   ├── source.xlsx            # 算出元の大阪歯科Excel（毎月Actionsが差し替え）
│   └── state.json             # 前回処理したファイル名・版・最終チェック（自動生成）
├── scripts/
│   └── update.py              # チェック→取得→集計→index.html更新
├── requirements.txt
└── .github/workflows/daily.yml
```

## データの流れ

1. `update.py` が一覧ページをスクレイプし、「歯科」のZIPリンクを取得
   （例 `…/kinki/2026.4_sisetukijun_sika.zip`。訂正版は `s2026.4_…` のように `s` が付く）。
2. 前回処理したファイル名と違えば、ZIPを取得して中の
   `{年月}_sisetukijun_osaka_sika.xlsx` を抽出。
3. 集計：
   - **母数** = A列「項番」のユニーク数（＝大阪府の歯科医療機関数）
   - **各施設基準** = N列「受理届出名称」× O列「受理記号」ごとに、届け出ている医療機関数（項番のユニーク数）
   - **届出率** = 届出医療機関数 ÷ 母数
4. `index.html` の `/*DATA:START*/ … /*DATA:END*/` を新しい集計で置換し、
   `data/source.xlsx` を更新。
5. 「最終チェック」日時は毎回更新するので、**毎日コミットが入ります**
   （これが cron の60日無効化を防ぐキープアライブも兼ねています）。

ダッシュボードの「Excelダウンロード」は、押した時だけ `data/source.xlsx` を取得して
集計シートと結合し、1ファイル（`届出状況_集計` ＋ `元データ` の2シート）で保存します。
ページ自体には元データを埋め込まないので軽いままです。

## セットアップ

1. このリポジトリを **public** で作成（public なら GitHub Actions の標準ランナーは無料・無制限）。
2. **Settings → Pages** で「Deploy from a branch」を選び、`main` / `root` を指定。
   → `https://<ユーザー名>.github.io/<リポジトリ名>/` で公開されます。
3. **Settings → Actions → General → Workflow permissions** を
   「Read and write permissions」にする（コミット/プッシュに必要）。
4. 初回は Actions タブから **Run workflow** を一度手動実行すると、現在公開中の最新版に揃います。

以降は毎日 06:00 JST 前後に自動チェック→更新されます。

## 無料アカウントでの運用について

- **public リポジトリ**：標準ランナーの Actions は無料・無制限。1日1回・数分なので費用は発生しません。
- private にする場合：Free プランは月2,000分。1日1回・数分なら月百数十分程度で十分収まります。
- 注意点（仕様）：
  - cron は UTC 基準・指定時刻ぴったりには動かず、混雑時 10〜30分ほど遅延することがあります（日次チェックなら問題なし）。
  - public のスケジュールworkflowは「60日間リポジトリ無活動」で自動無効化されますが、
    本構成は毎日コミットが入るため無効化されません。

## カスタマイズ

- 実行時刻：`.github/workflows/daily.yml` の `cron`（UTC）。06:00 JST 以外にしたい場合は時差換算して変更。
- 同じ仕組みで他府県（兵庫・京都など）や医科・薬局にも横展開可能
  （`update.py` の歯科ZIP判定と、zip内ファイル名のパターンを変えるだけ）。

## 注意

- 元データの内容（医療機関名・住所等）は近畿厚生局の公開情報です。再配布の可否・利用条件は出典元の規約をご確認ください。
- 訂正が「同じファイル名」で再公開された場合は検知できません（通常は `s` 付き等で名前が変わるため検知できます）。
