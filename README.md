# 近畿・北陸 歯科 施設基準 届出状況ダッシュボード

近畿厚生局が公開している「施設基準の届出受理状況（歯科）」をもとに、
近畿・北陸の府県ごとに歯科医療機関の **届出状況・届出率・推移** を表示する静的ダッシュボードです。
GitHub Actions が毎日1回データ更新をチェックし、新しい月のデータが出たら自動で反映します。

## 構成

```
.
├── index.html                  # ダッシュボード本体（軽量・自己完結）
├── data/
│   ├── prefectures.json        # メタ（版・現在日・最終チェック・府県一覧・公表月一覧）
│   ├── state.json              # 前回処理したファイル名・版・最終チェック（自動生成）
│   ├── current/<府県>.json     # 最新スナップショット（一覧表示用）
│   ├── history/<府県>.json     # 受理記号キーの時系列＋名称履歴（推移用）
│   └── source/<府県>.xlsx      # 算出元Excel（毎月Actionsが差し替え／DL同梱用）
├── scripts/
│   └── update.py               # チェック→取得→全府県集計→スナップショット追記
├── requirements.txt
└── .github/workflows/daily.yml
```

府県コード（ファイル名のromaji）: osaka 大阪 / hyogo 兵庫 / kyoto 京都 / shiga 滋賀 /
nara 奈良 / wakayama 和歌山 / fukui 福井 / ishikawa 石川 / toyama 富山。

## データの流れ

1. `update.py` が一覧ページをスクレイプし、「歯科」のZIPリンクを取得
   （例 `…/kinki/2026.5_sisetukijun_sika.zip`。訂正版は `s2026.5_…` のように `s` が付く）。
2. 前回処理したファイル名と違えば、ZIPを取得。**1つのZIPに全府県のxlsxが同梱**されているので、
   `{年月}_sisetukijun_{府県}_sika.xlsx` を全部抽出して府県ごとに集計する。
3. 集計（府県ごと）：
   - **母数** = A列「項番」のユニーク数（＝その府県の歯科医療機関数）
   - **施設基準の主キー = 受理記号（略称）**。記号ごとに、件数（行数）と
     届出医療機関数（項番のユニーク数）を出す。表示名はその月の名称（記号↔名称は月内で1:1）。
   - **施設基準の種類** = その府県で届出のある記号だけを数える（府県ごとに可変）。
   - **届出率** = 件数（または届出医療機関数）÷ 母数
4. 出力：
   - `current/<府県>.json` … 最新スナップショット（一覧用）
   - `history/<府県>.json` … **受理記号キー**の時系列に当月分を追記（推移用）。
     名称が変わった月は `names[]` に記録するので、**記号で連続性を保ったまま名称変更も追える**。
     掲載の無い月は点が増えないだけ（＝欠測。グラフ側で前後を破線でつなぐ想定）。
   - `prefectures.json` … 版・現在日・最終チェック・府県一覧・公表月一覧（推移の月軸）。
   - `source/<府県>.xlsx` … 元データ（ダウンロード同梱用）。
5. 「最終チェック」日時は毎回更新するので、**毎日コミットが入ります**
   （これが cron の60日無効化を防ぐキープアライブも兼ねています）。

### なぜ受理記号をキーにするか
施設基準は新設・廃止・名称変更が起きますが、**受理記号は名称変更があっても据え置かれることが多い**ため、
記号を主キーにすると名称変更をまたいで同一基準として時系列を連続させられます（マスタ管理が不要）。
実データでも、受理記号は空が無く、月内で記号↔名称が1:1でした。

## history JSON の形

```jsonc
{
  "versions": ["2026.5", "2026.6", ...],     // 公表された月（推移の月軸）
  "kigo": {
    "医療ＤＸ": {
      "name": "医療ＤＸ推進体制整備加算１",   // 最新の表示名
      "names": [                              // 名称履歴（変更があった月）
        {"v": "2026.5", "name": "医療ＤＸ推進体制整備加算"},
        {"v": "2026.7", "name": "医療ＤＸ推進体制整備加算１"}
      ],
      "series": [                             // 時系列（c=件数, u=届出医療機関数）
        {"v": "2026.5", "c": 1603, "u": 1601},
        {"v": "2026.6", "c": 1683, "u": 1680}
      ]
    }
  }
}
```

## セットアップ

1. このリポジトリを **public** で作成（public なら GitHub Actions の標準ランナーは無料・無制限）。
2. **Settings → Pages** で「Deploy from a branch」を選び、`main` / `root` を指定。
   → `https://<ユーザー名>.github.io/<リポジトリ名>/` で公開されます。
3. **Settings → Actions → General → Workflow permissions** を
   「Read and write permissions」にする（コミット/プッシュに必要）。
4. 初回は Actions タブから **Run workflow** を一度手動実行すると、現在公開中の最新版で
   全府県分の `current/` `history/` が生成されます。

以降は毎日 06:00 JST 前後に自動チェック→更新されます。

## 無料アカウントでの運用について

- **public リポジトリ**：標準ランナーの Actions は無料・無制限。1日1回・数分なので費用は発生しません。
- private にする場合：Free プランは月2,000分。1日1回・数分なら十分収まります。
- 注意点：
  - cron は UTC 基準で、混雑時 10〜30分ほど遅延することがあります（日次チェックなら問題なし）。
  - public のスケジュールworkflowは「60日間無活動」で自動無効化されますが、本構成は毎日コミットが入るため無効化されません。

## メモ：ダッシュボード（index.html）の接続
`index.html` は `data/prefectures.json` → 各 `current/<府県>.json`・`history/<府県>.json` を
読み込んで描画する想定です（推移は history を使用）。プレビュー用に大阪の実データを同梱しています。
